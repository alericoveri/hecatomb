htq2

A portable Quake II client

Hecatomb (a.k.a. htq2) is a Quake II client who's being intensely developed towards high portability

Serious decisions are being made. The main focus is portability, modernization and expansion capabilities, that means htq2 is not only portable but also extensible. With the power and stability of Yamagi Quake II, htq2 has already full support of 64-bit architectures, processors and operating systems.

Features

  • All the same goodies from YQ2
  • Solid codebase with porting capabilities
  • CMake build system fully automated for all supported platforms
  • Generic Hunk allocator (based on plain malloc/free) for easier porting
  • Hunk memory system debug through hunk_trace 1
  • Com_DPrintf as variadic macro (for easier tracing/debugging)
  • Signal handler for *nix systems with backtrace (if supported by target platform)

Supported architectures

  • AMD64 (a.k.a. x86_64)
